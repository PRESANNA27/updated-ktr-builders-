## SYSTEM 1 · COMPLETE VISUAL IDENTITY

### 1.1 Brand Positioning

KTR Builders is not a contractor. KTR Builders is an engineering firm led by a government-registered civil engineer with 28 years of experience. The visual identity must make this distinction unmistakable.

**Brand Archetype:** The Expert / The Authority  
Not the friendly neighbourhood builder. Not the low-price option. The expert you trust with your most expensive decision. Think: a senior surgeon's practice, a chartered accountant's office, a prestigious law firm — precision, experience, credentials, calm confidence.

**Visual Tone:** Premium without arrogance. Authoritative without coldness. Warm without informality.  

**Three Words that Define Every Design Decision:**
1. **Trustworthy** — The design must signal reliability before a word is read
2. **Expert** — The design must communicate engineering precision
3. **Local** — The design must feel rooted in Karaikal, not imported from Bangalore

### 1.2 Logo Usage Rules

The KTR Builders logo must always appear:
- With minimum clear space equal to the height of the letter "K" on all four sides
- Never stretched, skewed, or rotated
- Never placed on a busy photographic background without a frosted/solid backing
- In two approved variants: Full colour (navy + gold) and reversed (white + gold on dark backgrounds)
- Minimum size: 120px wide on screen, never smaller

**Logo Placement Priority:**
- Primary: Top-left of navigation bar on every page
- Secondary: Footer column 1, centred
- Never repeated elsewhere on the same page

### 1.3 Brand Voice

**In headings:** Confident, declarative, specific. *"Engineer-Led Construction Since 1999"* — never vague: *"Quality Construction Services"*

**In body copy:** First-person authority. Er. Sasi Kumar speaks directly, with professional specificity. *"I specify M30 grade concrete for every column in Karaikal's coastal conditions — not because it costs more, but because the soil requires it."*

**In CTAs:** Direct, personal, low-pressure. *"Talk to Er. Sasi Kumar"* — not *"Submit Enquiry Form"*

**Never use:**
- Superlatives without proof: "Best in India," "World-class"
- Vague promises: "Quality assured," "100% satisfaction"
- Corporate jargon: "Solutions," "Deliverables," "Synergies"
- Pressure language: "Limited time offer," "Act now"

The palette is built on two anchors: **deep navy** (authority, trust, engineering precision) and **warm gold** (value, craft, quality). These two colours are the most used combination in premium engineering and financial services globally — they communicate exactly what KTR Builders needs to communicate.

Supporting neutrals keep the design breathable. The palette is deliberately restrained — fewer colours, used consistently, create a more premium impression than many colours used inconsistently.

### 2.3 Color Usage Rules

**Gold is earned.** Gold is used only for:
- Primary CTA buttons
- Interactive hover borders on cards
- Statistical numbers (the "28+" in "28+ Years")
- Section divider bars (the 3px horizontal rule under section eyebrows)
- Active navigation state

Gold is never used for:
- Large background fills (except the KPA strip marquee)
- Long paragraphs of text
- Purely decorative elements that don't require user attention
- More than 15% of any single visible viewport

**Navy carries the brand.** Navy is used for:
- All dark section backgrounds
- Navigation bar (frosted dark on scroll)
- Footer
- Dark card variants
- Text on gold backgrounds

**White creates breathing room.** The majority of the website uses white and slate-50 backgrounds. Dark sections (navy) are used strategically for emphasis — no more than 3 consecutive dark sections.

### 2.4 Color Accessibility

All text/background combinations must meet WCAG 2.1 AA minimum (4.5:1 contrast):

```
Body text (Slate 700 on White):    7.4:1    ✓ PASS
Headings (Slate 900 on White):     16.1:1   ✓ PASS
Body text (White on Navy 800):     12.2:1   ✓ PASS
Gold text (Gold 700 on White):     4.7:1    ✓ PASS
Body text (White on Gold 600):     2.8:1    ✗ FAIL — never put body text on gold
Gold accents on Navy 800:          6.1:1    ✓ PASS
```

**Rule:** Gold 600 and brighter are NEVER used as text backgrounds for body copy. Only Navy 800 and Slate 700+ are used on gold backgrounds as text.

### 2.5 Dark Mode

Dark mode respects `@media (prefers-color-scheme: dark)`. The dark mode colour shifts are:

```
Page background:     White → Navy 950 (#070F1F)
Section alt bg:      Slate 50 → Navy 900 (#0A1628)
Card background:     White → Navy 800 (#0F1F3D)
Primary body text:   Slate 700 → Slate 200
Secondary text:      Slate 600 → Slate 400
Dividers/borders:    Slate 200 → Navy 700
Input background:    White → Navy 800
Input border:        Slate 300 → Navy 600
Input text:          Slate 900 → Slate 100
Gold accents:        Unchanged (Gold 600 works on both light and dark)
CTA buttons:         Unchanged (gold buttons work on both)
```

Dark mode never inverts photography. Project photos, founder photo, and certificate photos always render true to life.

## SYSTEM 3 · TYPOGRAPHY SYSTEM

### 3.1 Typeface Philosophy

Two typefaces only. The single most powerful way to achieve a premium feel is typographic restraint: one display face, one body face, used consistently throughout every page.

**Display Face: Cormorant Garamond**
- Use: All H1, H2, H3 headings; large pull quotes; statistical numbers
- Why: Cormorant Garamond carries the weight of centuries of professional credibility. It is the typeface of architects' drawings, engineering firms, law offices, and premium real estate developers globally. In the context of a Karaikal construction website, it instantly communicates a different class of business.
- Google Fonts: `Cormorant Garamond:ital,wght@0,500;0,600;0,700;1,500;1,600`
- Fallback: Georgia, 'Times New Roman', serif

**Body Face: Inter**
- Use: All body text, navigation labels, UI elements, form inputs, captions, buttons, badges
- Why: Inter is designed specifically for screen legibility. It is the most legible sans-serif at small sizes on mobile screens. It is neutral, professional, and fast to load.
- Google Fonts: `Inter:wght@300;400;500;600;700`
- Fallback: -apple-system, BlinkMacSystemFont, 'Segoe UI', sans-serif

**Font Loading Rule:** Both fonts load non-blocking using `media="print" onload="this.media='all'"` technique. `font-display: swap` prevents invisible text flash.

### 3.2 Type Scale

All sizes use `clamp()` for fluid responsive scaling between mobile (min) and desktop (max).

```
DISPLAY / HERO SIZES:
─────────────────────────────────────────────────────────────────────
fs-hero:     clamp(2.75rem, 6.5vw, 5rem)      44–80px   H1 Hero
fs-display:  clamp(2.25rem, 5vw, 4rem)        36–64px   Large feature text
fs-hero-sub: clamp(1.125rem, 2vw, 1.375rem)  18–22px   Hero subheadline

HEADING SIZES:
─────────────────────────────────────────────────────────────────────
fs-h1:       clamp(2rem, 4vw, 3rem)           32–48px   Page H1s (non-hero)
fs-h2:       clamp(1.625rem, 3vw, 2.25rem)   26–36px   Section H2s
fs-h3:       clamp(1.25rem, 2.2vw, 1.625rem) 20–26px   Subsection H3s
fs-h4:       clamp(1.125rem, 1.8vw, 1.25rem) 18–20px   Card titles, sidebar H4
fs-h5:       1rem                             16px      Minor headings

BODY SIZES:
─────────────────────────────────────────────────────────────────────
fs-body-lg:  clamp(1.0625rem, 1.5vw, 1.125rem)  17–18px  Lead paragraphs
fs-body:     1rem                               16px     Body text (fixed, ≥16px iOS rule)
fs-body-sm:  0.9375rem                          15px     Captions, secondary text
fs-caption:  0.875rem                           14px     Image captions, form labels
fs-micro:    0.8125rem                          13px     Fine print, legal text
fs-badge:    0.75rem                            12px     Credential badges, tags

SPECIAL:
─────────────────────────────────────────────────────────────────────
fs-stat:     clamp(2.5rem, 5vw, 4rem)          40–64px  Statistics (28+, 60+)
fs-stat-sub: clamp(0.875rem, 1.2vw, 1rem)      14–16px  Stat label ("Years of Experience")
fs-eyebrow:  0.8125rem                          13px     Section eyebrow labels (all-caps)
fs-quote:    clamp(1.25rem, 2.2vw, 1.625rem)   20–26px  Pull quotes, testimonials
```

### 3.3 Typographic Hierarchy Rules

**Eyebrow labels** (above H2s): Always Inter, 0.8125rem, font-weight 600, letter-spacing 0.12em, all uppercase, gold colour. Example: `ABOUT KTR BUILDERS`

**H2 section headings**: Always Cormorant Garamond, weight 600, Slate 900 on white backgrounds, White on dark navy backgrounds.

**Lead paragraphs**: Inter, 1.0625rem, font-weight 400, line-height 1.75, Slate 700. First paragraph of each section is always slightly larger than body text.

**Body paragraphs**: Inter, 1rem (16px minimum — never below), font-weight 400, line-height 1.7, Slate 700. Maximum line length: 68 characters (680px column max on desktop). Never wider. Long lines destroy readability.

**Statistics**: Cormorant Garamond, weight 700, the number in Gold 600, the unit ("+", "★") also Gold 600, the label below in Inter 600 uppercase Slate 600.

**Pull quotes / Founder quotes**: Cormorant Garamond italic, weight 500, font-size fs-quote, Gold 700, preceded by a 3px gold left border, 32px left padding.

**CTA button text**: Inter, font-weight 600, letter-spacing 0.02em, never italic.

**Form labels**: Inter, 0.875rem, font-weight 500, Slate 700. Always above the input, never inside.

### 3.4 Line Height & Spacing

```
Heading line-height:    1.2   (tight, impactful for large display text)
Subheading line-height: 1.3
Body line-height:       1.7   (generous, increases readability and premium feel)
Lead paragraph:         1.75
Caption:                1.5
Button text:            1     (single line only)
```

## SYSTEM 4 · SPACING SYSTEM

### 4.1 Base Unit

The spacing system is built on an **8px base unit**. Every spacing value is a multiple of 8px. This creates visual rhythm and mathematical harmony across the entire design.

```
SPACING SCALE:
─────────────────────────────────────────────────────────────────────
space-1:   4px    (0.5×) — Internal component micro-gaps
space-2:   8px    (1×)   — Base unit, tight internal gaps
space-3:   12px   (1.5×) — Between icon and label, form element gaps
space-4:   16px   (2×)   — Between related elements within a component
space-5:   24px   (3×)   — Between components within a section
space-6:   32px   (4×)   — Section internal spacing, card internal padding
space-7:   40px   (5×)   — Between major elements within a section
space-8:   48px   (6×)   — Mobile section vertical padding
space-9:   64px   (8×)   — Tablet section vertical padding
space-10:  80px   (10×)  — Desktop section vertical padding
space-11:  96px   (12×)  — Large section vertical padding (desktop)
space-12:  120px  (15×)  — Maximum section vertical padding
```

### 4.2 Section Vertical Padding

Section vertical padding (top and bottom) uses fluid scaling:

```
section-v:      clamp(64px, 10vw, 120px)   Standard section top/bottom padding
section-v-sm:   clamp(48px, 7vw, 80px)    Compact sections (stats strip, trust bar)
section-v-lg:   clamp(80px, 12vw, 160px)  Featured sections (founder, hero)
```

### 4.3 Component Internal Spacing

```
Card padding (large):    40px 40px (desktop) → 24px 20px (mobile)
Card padding (standard): 32px 32px (desktop) → 20px 16px (mobile)
Card padding (compact):  20px 24px (desktop) → 16px 16px (mobile)

Button padding (large):  18px 40px
Button padding (standard): 14px 32px
Button padding (small):  10px 20px

Input padding:           14px 16px (height minimum 52px on mobile)
Textarea padding:        14px 16px
```

## SYSTEM 6 · BUTTON SYSTEM

### 6.1 Button Philosophy

Every button is a trust signal as much as a conversion element. Buttons must communicate: *"This is safe to press. This is the right action. This person will respond."*

### 6.2 Button Variants

**PRIMARY BUTTON (Gold) — Main conversion action**

```
PURPOSE: The singular most important action on any page — "Get Free Consultation,"
         "Get Quote," "Contact Us." Only one primary button per viewport.

VISUAL:
  Background:        Gold 600 (#B8892A)
  Text:              White (#FFFFFF), Inter 600, 0.02em letter-spacing
  Border-radius:     6px (not fully rounded — authority, not playfulness)
  Padding:           18px top/bottom, 40px left/right (large) / 14px 32px (standard)
  Min-height:        56px (desktop), 52px (mobile) — generous tap target
  Box-shadow:        0 4px 16px rgba(184, 137, 42, 0.35)
  
HOVER STATE:
  Background:        Gold 400 (#E8B84B)
  Transform:         translateY(-2px)
  Box-shadow:        0 8px 24px rgba(184, 137, 42, 0.45)
  Transition:        all 0.25s cubic-bezier(0.4, 0, 0.2, 1)

ACTIVE/PRESSED:
  Transform:         translateY(0)
  Box-shadow:        0 2px 8px rgba(184, 137, 42, 0.3)

DISABLED:
  Background:        Slate 300
  Text:              Slate 500
  Cursor:            not-allowed

MOBILE BEHAVIOR:
  Full width on screens < 640px (unless inside button group)

- Disabled state: `aria-disabled="true"`, `tabindex="-1"`

## SYSTEM 7 · FORM SYSTEM

### 7.1 Form Philosophy

Forms are the moment of highest friction in the conversion journey. Every form element must reduce anxiety, not add to it. The goal is for the user to feel: *"This is easy. They only need a little information. I can trust what I provide here."*

### 7.2 Input Field Design

```
STANDARD INPUT:
─────────────────────────────────────────────────────────────────────
Height:          52px minimum (mobile thumb-friendly, prevents iOS zoom)
Font-size:       1rem (16px — critical to prevent iOS auto-zoom)
Font:            Inter 400, Slate 800
Padding:         14px 16px
Background:      White
Border:          1.5px solid Slate 300
Border-radius:   6px
Label:           Above input, Inter 500, 0.875rem, Slate 700
Label gap:       6px between label and input
Placeholder:     Slate 400, italic
Transition:      border-color 0.2s

FOCUS STATE:
  Border:        1.5px solid Navy 600
  Outline:       3px solid rgba(30, 58, 110, 0.15) (soft glow, not harsh)
  Background:    White

VALID STATE (after user input):
  Border:        1.5px solid #1A6B3A (success green)
  Trailing icon: Small checkmark, success green

ERROR STATE:
  Border:        1.5px solid #B91C1C (error red)
  Trailing icon: Warning icon, red
  Error message: Below input, Inter 400, 0.8125rem, #B91C1C

DARK MODE:
  Background:    Navy 800
  Border:        1.5px solid Navy 600
  Text:          Slate 100
  Placeholder:   Slate 500
```

### 7.3 Form Layouts

**Simple Enquiry Form (sidebar / modal)**

  Back + Next

STEP 4 — Contact

  Phone: tel input (primary — explicitly labelled "Mobile Number")
  Best time to call: [Morning 9–12] [Afternoon 12–5] [Evening 5–7]

PARTIAL COMPLETION BEHAVIOR:

  If user exits at step 2+, exit-intent popup shows: "Get the quote you started →"
  Not shown on page load — only on exit from /get-quote/
```

### 7.4 Form Micro-Copy

## SYSTEM 12 · DARK MODE STRATEGY

### 12.1 Implementation Approach

Dark mode is implemented via CSS Custom Properties and `@media (prefers-color-scheme: dark)`. The user's operating system preference is automatically respected. No JavaScript required for basic dark mode.

A manual toggle is NOT provided. Reason: A manual toggle adds UI complexity and JavaScript dependency. OS-level dark mode respects the user's stated preference.

### 12.2 Dark Mode Exceptions

These elements **never** change in dark mode:
- Photography (project photos, founder photo, certificate photos) — always true colour
- Gold accent colours (Gold 600, Gold 500) — work on both light and dark
- Google Maps embed — browser-handled, not the site's dark mode
- WhatsApp green button — brand colour, unchanged

### 12.3 Dark Mode Quality Test

Before launch, dark mode must be tested against this checklist:
```
□ All text remains readable (4.5:1 minimum contrast maintained)
□ No white flashing on initial load
□ Forms remain usable (inputs visible, labels readable)
□ Gold CTAs remain prominent (not washed out)
□ Cards visible against dark backgrounds (sufficient border/shadow)
□ Footer visible (dark on dark must be differentiated by subtle border)
□ Images look natural (not colour-shifted)
□ Navigation legible on scroll (frosted effect still works)
```

## SYSTEM 13 · ANIMATION STRATEGY

### 13.1 Animation Philosophy

Animations at KTR Builders have one purpose: **to direct attention to proof of expertise and trust signals**. Animations never exist for visual entertainment. Every animation must earn its place by serving a UX purpose.

**Rules:**
1. No animation that slows down content display (all animations on already-loaded elements)
2. No animation on hero headline — the headline must be instantly readable
3. No parallax scrolling (causes motion sickness, hurts performance)
4. No auto-playing video without explicit user action (except background hero loop if subtle)
5. Respect `prefers-reduced-motion` — all animations are disabled for users who request it

### 13.2 Animation Inventory

**Scroll Reveal (IntersectionObserver — no library)**
```
TRIGGER:     Element enters viewport (10% threshold)
APPLIES TO:  Section headlines, stat cards, service cards, project cards, testimonials

ANIMATION VARIANTS:
  Fade Up:      opacity: 0 → 1, translateY: 20px → 0, duration 0.6s ease-out
  Fade In:      opacity: 0 → 1, duration 0.5s ease-out (for images)
  Scale In:     opacity: 0 → 1, scale: 0.97 → 1, duration 0.5s ease-out (for cards)
  
STAGGER:      When multiple cards reveal, delay increases by 0.1s per card:
              Card 1: 0s, Card 2: 0.1s, Card 3: 0.2s — creates "cascade" effect

ONCE ONLY:    Animation fires once per session — no re-trigger on scroll up/down
```

**Statistics Counter (IntersectionObserver)**
```
TRIGGER:     Stats section enters viewport
APPLIES TO:  28+, 60+, 100+, 5.0 — the numerical values
ANIMATION:   Number counts up from 0 to final value over 1.5–2s
EASING:      easeOut — fast at start, slows at the end
PURPOSE:     Makes the user "watch" the numbers reach their value,
             creating a micro-moment of engagement with the proof
FALLBACK:    If JS is disabled, shows final value immediately — no broken display
```

**Navigation Transition (CSS only)**
```
TRIGGER:     Page scroll > 80px
APPLIES TO:  Navigation bar
ANIMATION:   background-color, backdrop-filter, box-shadow transition, 0.35s ease
             transparent → frosted navy
```

**Button Hover (CSS only)**
```
TRIGGER:     Mouse hover
ANIMATION:   translateY(-2px), shadow increase, colour brighten
DURATION:    0.25s cubic-bezier(0.4, 0, 0.2, 1)
```

**Card Hover (CSS only)**
```
TRIGGER:     Mouse hover
ANIMATION:   translateY(-4px to -6px), shadow increase, border colour change to gold
DURATION:    0.3s cubic-bezier(0.4, 0, 0.2, 1)
IMAGE ZOOM:  Scale 1.04 on image within card, overflow hidden on card
```

**Floating WhatsApp Button**
```
TRIGGER:     Page load, after 3 seconds
ANIMATION:   Scale from 0 to 1 with bounce: cubic-bezier(0.34, 1.56, 0.64, 1)
PULSE:       Subtle ring pulse every 8s (scale 1 → 1.2, opacity 1 → 0)
             Purpose: Draws attention without being annoying
```

**Mobile Drawer**
```
TRIGGER:     Hamburger tap
ANIMATION:   Clip-path circle expand from hamburger position, 0.4s ease
CLOSE:       Reverse animation, 0.35s ease
```

**Mega Dropdown**
```
TRIGGER:     Hover on "Services" or "Areas"
ANIMATION:   opacity 0→1, translateY -8px→0, 0.25s ease
```

**Gold underline hover (nav links)**
```
TRIGGER:     Hover
ANIMATION:   scaleX transform 0→1 from left, 0.2s ease, 2px gold underline
```

### 13.3 Performance Budget for Animation

```
No GSAP, no Animate.css, no Motion library.
All animations: CSS transitions + CSS keyframes + Vanilla JS IntersectionObserver.
Total animation JavaScript: < 2KB gzipped.
GPU-accelerated only: transform and opacity — never animate width, height, top, left.
```
